# Фабрика гипотез (Норникель, AI Science Hack)

Прототип системы, которая по данным о хвостах обогатительных фабрик находит
**зоны неопределённости** и формирует **проверяемые гипотезы** для снижения
потерь элемента 28 (Ni) и элемента 29 (Cu).

## Задача и заказчик

- **Заказчик / контекст:** кейс «Фабрика гипотез», Норникель AI Science Hack.
- **Бизнес-цель:** уменьшить потери ценных элементов в хвостах флотации.
- **Задача системы:** из разнородных отчётов по хвостам автоматически выделить,
  где потери технологически возвратны и плохо изучены, и предложить гипотезы
  с минимальным проверочным экспериментом — ранжированные по интерпретируемым
  критериям.

## Данные

Вход — 4 реальных примера фабрик (`01_Data/raw/.../Задача 1/`), сведённые в
`01_Data/processed/nornickel_real_case_data_summary.json`:

| Артефакт | Объём |
|---|---|
| Фабрики | 4 (КГМК, НОФ Вкр, НОФ мед, ТОФ) |
| Строки-факты в таблицах хвостов | 117 |
| Наблюдения (факт × элемент 28/29) | **230** |
| Факты о потерях по классам (`EvidenceClaim` TAIL-*) | **94** |
| Экспертные гипотезы из `.docx` (КГМК 5 / НОФ Вкр 6 / НОФ мед 8 / ТОФ 8) | **27** |

Источники: таблицы `Хвосты *.xlsx`, экспертные `Гипотезы *.docx` (лежат внутри
таблиц Word), регламенты и схемы флотации (изображения), справочная база знаний
`nornickel_real_case_knowledge_base.md`. Экспертные гипотезы обезличены как
«элемент 28/29» в исходных данных.

## Архитектура pipeline

Сквозной конвейер — 11 шагов, от приёма данных до экспорта. Детерминированный,
без LLM (`generation/llm_client.py` — заглушка-хук на будущее).

**Диаграммы:** [BPMN pipeline](docs/pipeline_bpmn.md) · [ER-онтология](docs/ontology_er.md)

Приём данных → сборка документов → чанкинг → факты/claims → сущности →
матрица изученности → граф знаний → **зоны неопределённости** → генерация
гипотез → скоринг/ранжирование → экспорт.

Результат на дефолтном KPI: **133 сущности**, **28 зон** 4 типов
(`coarse_locked_loss` 8, `missing_process_link` 6, `contradiction` 6,
`expert_unvalidated` 8), **8 ранжированных гипотез** (из top-8 зон по приоритету).

## Что делает каждый модуль

| Модуль | Назначение |
|---|---|
| `data_loaders/real_case_loader.py` | Загрузка сводки, построение наблюдений, документов и claims. |
| `data_loaders/hypotheses_docx_parser.py` | Парсинг экспертных гипотез из `.docx` (в т.ч. из таблиц Word). |
| `data_loaders/tailings_xlsx_parser.py`, `image_registry.py` | Разбор таблиц хвостов и реестр изображений (схемы/регламенты). |
| `ingestion/chunking.py`, `parsers.py` | Нарезка документов на чанки. |
| `extraction/claim_extractor.py`, `entity_extractor.py` | Извлечение фактов и сущностей (фабрика, класс, элемент, оборудование…). |
| `analysis/tailings_analyzer.py` | Матрица изученности + поиск и приоритизация зон неопределённости. |
| `analysis/coverage_analyzer.py`, `contradiction_analyzer.py`, `gap_analyzer.py`, `indirect_link_analyzer.py`, `uncertainty_mapper.py` | Аналитика для legacy-режима (общий текстовый корпус). |
| `graph/graph_builder.py`, `graph_viz.py` | Граф знаний из claims. |
| `generation/hypothesis_generator.py`, `templates.py` | Генерация гипотез (что менять, механизм, мин. эксперимент, критерии). |
| `analysis/experiment_planner.py` | План минимального проверочного эксперимента. |
| `scoring/scorer.py`, `profiles.py` | Взвешенная оценка и ранжирование (6 профилей весов). |
| `analysis/expert_match.py` | Сопоставление сгенерированных гипотез с экспертными. |
| `export/export_json.py`, `report_docx.py` | Выгрузка JSON / CSV / DOCX. |
| `src/app.py` | Streamlit-интерфейс. |

## Как запустить

Из папки `Nornickel-hypothesis-factory/`:

```bash
python -m venv .venv && source .venv/bin/activate     # Python 3.12
pip install -r requirements.txt

# тесты
python -m pytest -q                                   # ожидается: 15 passed, 6 subtests

# сопоставление сгенерированных и экспертных гипотез
PYTHONPATH=src python scripts/compare_hypotheses.py

# интерактивное приложение
python -m streamlit run src/app.py                    # http://localhost:8501
```

> Требуется `python-docx` (в `requirements.txt`) — иначе экспертные гипотезы не
> парсятся и числа падают до 103 сущностей / 20 зон.

## Сопоставление с экспертными гипотезами

`scripts/compare_hypotheses.py` сравнивает 8 сгенерированных гипотез с 27
экспертными по прозрачному пересечению технологических концептов (не ML-модель):

- **2 из 8** совпадают по конкретной технологической мере (доизмельчение +
  классификация в грубых классах — совпадает с экспертными идеями про
  доизмельчение, замену классификаторов на гидроциклоны и т.п.);
- **6 из 8** пересекаются тематически (общая идея «привязать пик потерь к узлу
  схемы / оборудованию»).

Метрика намеренно грубая и объяснимая; таблица соответствий печатается скриптом.
